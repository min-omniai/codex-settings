# Unity Teaching Patterns

## Full Lecture Template

Use this shape for 60-120 minute classes:

1. Hook: a concrete in-game problem students recognize.
2. Mental model: explain the Unity concept before showing code.
3. Minimal demo: one scene, one script, one visible result.
4. Guided practice: students modify one behavior safely.
5. Debug moment: show one common failure and how to inspect it.
6. Independent task: small feature with clear success criteria.
7. Review: connect the lesson to production concerns.

## Assignment Rubric

Score against:
- Correct behavior
- Unity lifecycle usage
- Readability
- Inspector/prefab setup
- Error handling
- Performance awareness
- Ability to explain the solution

## Student Feedback Tone

Use a coaching tone:
- Start with one thing that works.
- Name the highest-impact issue.
- Explain the reason in beginner-friendly language.
- Give a hint before giving corrected code.
- Keep feedback actionable and short enough to apply immediately.
